## 一、项目简介

基于 **TM4C1294NCPDT** (Cortex-M4, 20MHz) 的嵌入式时钟系统。固件用 **ARM Compiler 5 / Keil MDK** 构建 (~38 KB Flash)，所有 MCU 代码合并为单一 **main.c** (4718 行，19 源文件拼装)。PC 上位机用 **Python 3.11 / PyQt5 / pyserial** (~3000 行 + ~620 行 QSS)。两者通过 **UART 115200-8-N-1** ASCII 协议通信。

自主扩展：**开机旋律** (See You Again, 68 音符, 非阻塞 PWM) + **步进电机** (28BYJ-48, 8 拍半步, 1 RPM)。

## 二、硬件平台与架构

| 组件 | 型号 | 连接 |
|------|------|------|
| MCU | TM4C1294NCPDT, 20MHz | — |
| 数码管/LED | TCA6424 (0x22) + PCA9557 (0x18) | I2C0: PB2/PB3 |
| 按键 K1-K8 / USER1-2 | TCA6424 PORT0 / PJ0-PJ1 上拉 | I2C0 / GPIO |
| 蜂鸣器 | 无源 PK5 (M0PWM7) | PWM0 Gen3 |
| 步进电机 | 28BYJ-48 PF0-PF3 (8拍半步) | Timer0A ~68Hz |
| RTC / UART | Hibernate 32kHz / PA0-PA1 | 内部 / UART0 115200 |

### 固件结构 — 单文件 main.c (4718 行)

19 源文件按依赖顺序拼接，去重头文件，static 隔离：

```
[头文件] stdint/stdbool + 19 driverlib/inc 引用 → [内联.h] 类型/结构体/函数声明 →
S1 app_state(全局g) S2 delay S3 hibernate S4 timebase S5 i2c S6 pwm S7 stepper
S8 uart S9 board(DevicesInit) S10 datetime S11 melody(68音符) S12 alarm(200ms蜂鸣)
S13 display(7SEG+LED+事件) S14 boot_seq S15 button S16 command(20+命令)
S17 app(主循环) S18 callback(3 ISR) → main(): DevicesInit→App_Init→while(1)App_RunOnce
```

### 主循环与中断

```
App_RunOnce: Melody→Alarm→[init?启动动画:按键/显示/LED]→Beep超时→ProcessUartCommand
频率: 7SEG 500Hz · 缓冲 1Hz · 流移 2/3.3Hz · 无阻塞
```

| 中断 | 频率 | 职责 |
|------|------|------|
| SysTick | 1kHz | tick++, **时钟进位**(ss→mm→hh→day→mon→year,闰年感知), 按键20ms去抖, 5s超时 |
| UART0 | 按需 | 字符→rx_buf, `\r\n`触发cmd_state, 多命令`\0`分隔缓冲 |
| Timer0A | ~68Hz | 步进电机8拍序列 (4096拍/圈, 1RPM) |

全局状态 `app_state_t g` 含 7 子结构体(clock/disp/timer/uart/in/motor/rtc)。ARMCC5 全字段显式 `=false/=0`。

## 三、UART 协议

UART0 115200-8-N-1, ASCII, 大小写不敏感 · 空格/Tab容忍 · 最小匹配缩写(`HOU`→`HOUR`)。

### 命令表 (PC→MCU)

| 命令 | 说明 |
|------|------|
| `*SET:DATE/TIME YEAR/MONTH/DATE...` | 日期/时间 (10种字段组合, 自动调整) |
| `*SET:ALARM ...` / `OFF` | 闹钟; OFF设alm_hh=25哨兵关闭 |
| `*SET:DISPLAY ON/OFF` / `*SET:FORMAT LEFT/RIGHT` | 开关 / 方向(shift_mode+reversed) |
| `*SET:MSG <text>` (≤32B) | 临时消息 (≤8字符静态2.5s, >8滚动300ms/位) |
| `*SET:LED <hex2>` / `*SET:WEATHER <hex2>` | LED接管(00退出) / 天气LED5-7(掩码0xE0,不接管) |
| `*SET:BEEP <ms>` (10-5000) / `*SET:MODE NIGHT/DAY` | 远程蜂鸣2400Hz / 夜间模式 |
| `*SET:KEY FUNC/SHIFT/.../USER1/USER2` | 虚拟按键注入 |
| `*GET:DATE/TIME/ALARM/DISPLAY/FORMAT/MOTOR` | 查询 |
| `*MOTOR:START/STOP/FWD/REV` / `*RST` / `*NTP SYNC` / `*PING` / `HELP` / `INIT` | 电机/系统 |

响应: `OK`/`OK <value>` 或 `ERROR SYNTAX`/`PARAM`/`RANGE`/`BUSY`

### 事件 (MCU→PC)

`*EVT:DISP <8chars> <dpHex>` (变化+1Hz心跳) · `*EVT:LED <hex2>` · `*EVT:KEY USER1/USER2/EXT`  
`*EVT:MODE FLOWING/DATE/TIME/ALARM/NIGHT/DAY` · `*EVT:ALARM[/OFF]` · `*EVT:EDIT DATE/TIME/ALARM <val>`  

FORMAT RIGHT: 字符+DP位图双向镜像 · 6字符 `CenterContent()` 居中, 两侧`_`

### LED 状态

| LED | 位 | 含义 |
|-----|---|------|
| 0 | 0x01 | 1Hz心跳 | | 3 | 0x08 | UART活动(100ms) |
| 1 | 0x02 | 闹钟 常亮/200ms闪烁 | | 4 | 0x10 | NTP对时完成 |
| 2 | 0x04 | 编辑模式 | | 5-7 | 0xE0 | 天气: 晴0x80/多云0x40/雾0x60/雨0x20/雪0xA0/雷暴0xC0/未知0xE0 |

## 五、自主功能一 — 开机旋律 "See You Again"

**动机:** 利用已有PWM蜂鸣器增强开机体验, 无需额外硬件。

**设计:** 非阻塞状态机 · 文件域static变量 · `{freq_hz, duration_ms}` 音符(freq=0=休止) · 23ms staccato · 68音符(34旋律+34休止), 587-1175Hz

```c
void Melody_Update(void) {
    if (!s_active) return;
    const note_t *note = &see_you_again[s_note_index];
    if ((g.timer.tick - s_note_start) < note->duration_ms) return;
    if (++s_note_index >= MELODY_NUM_NOTES) { PWMStop(); s_active = false; return; }
    note = &see_you_again[s_note_index];
    s_note_start = g.timer.tick;
    note->freq_hz ? PWMStart(note->freq_hz) : PWMStop();
}
```

**演示:** 上电→启动动画→蜂鸣器播放(~15s)→自动结束 · 不阻塞时钟/按键/命令 · 与闹钟PWM独立

## 六、自主功能二 — 步进电机 28BYJ-48

**动机:** 开发板配有28BYJ-48接口(PF0-PF3), 课程未要求, 作为附加硬件展示。

**设计:** 8拍半步 `{0x01,0x03,0x02,0x06,0x04,0x0C,0x08,0x09}` · 4096拍/圈 · `(20MHz×60)/(4096×1)-1≈292967`→Timer0A ~68Hz=1RPM · PF0 `GPIO_LOCK_KEY` 解锁

```c
void StepperTimerTick(void) {
    TimerIntClear(TIMER0_BASE, TIMER_TIMA_TIMEOUT);
    if (!g.motor.running) return;
    GPIOPinWrite(GPIO_PORTF_BASE, STEPPER_PINS,
                 stepper_sequence[g.motor.position & 0x07]);
    if (g.motor.direction == 0)
        { if (++g.motor.position >= 4096) g.motor.position = 0; }
    else
        { if (g.motor.position-- == 0) g.motor.position = 4095; }
}
```

**演示:** 上电1RPM恒转 · PC面板 START/STOP/FWD/REV · `*GET:MOTOR`查询 · 中断驱动不阻塞主循环

## 七、难点攻克

| # | 问题 | 根因 | 修复 |
|---|------|------|------|
| 1 | ARMCC5不零初始化→开机逆序/闪烁 | 未显式赋值嵌套struct字段为随机值 | 全字段`=false/=0`, FinishBootSequence二次确认 |
| 2 | O1+破坏7SEG栈数组→乱码 | 循环展开寄存器分配不稳 | ~50个显式if-else替代循环+数组 |
| 3 | 多命令`\0`覆盖尾字符→丢字节 | pre-inc使rx_len指尾字符 | `\r\n`处理区分cmd_state: 追加先`++rx_len` |
| 4 | `*EVT:DISP`空格破坏PC解析 | split()拆分payload内空格 | 固定位置 chars[0:8] dp[9:11] |
| 5 | ReverseDpBitmap bit7丢失 | `1<<i→1<<(6-i)`未覆盖i=7 | bit7→bit0单独处理 |
| 6 | `OK ERROR PARAM`畸形响应 | 先发OK再校验字段 | 预扫描合法字段后发前缀 |
| 7 | PF0 NMI锁定→GPIO失败 | TM4C1294 PF0默认NMI | GPIO_LOCK_KEY解锁+GPIO_CR提交 |
| 8 | `*GET:*`被ERROR BUSY误拦 | 未区分只读/修改命令 | 放行*GET:* / *PING / HELP / *RST |
| 9 | 'C'显示为小写c | seg7[12]=0x58缺a/f段 | 改0x39(段a,d,e,f), 与PC端一致 |
| 10 | USER1/2虚拟按键无响应 | suppress阻止*EVT:KEY上报 | 移除USER1/2的suppress |

## 八、亮点总结

**固件:** 单文件4718行(19模块拼装) · 堆=0全静态 · 非阻塞全链路(旋律/闹钟/显示/命令协作) · 编辑-提交-回滚 · ISR inline进位(闰年感知) · 7SEG反向映射(CharForSegment) · LED变化检测双缓冲 · *PING手动itoa
